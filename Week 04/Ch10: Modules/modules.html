Modules

Modules provide structure to bigger programs by separating the code into pieces with clear
interfaces and dependencies. The most widely used approach to bolted-on JavaScript modules is
called CommonJS modules. Node.js uses it and is the system used by most packages on NPM.

Roads module

// Add dependencies and exports
const roads = [
"Alice's House-Bob's House", "Alice's House-Cabin",
"Alice's House-Post Office", "Bob's House-Town Hall",

The solution is:

const {buildDesign} = require("./design");
const roads = [
"Alice's House-Bob's House", "Alice's House-Cabin",
"Alice's House-Post Office", "Bob's House-Town Hall",
"Daria's House-Ernie's House", "Daria's House-Town Hall",
"Ernie's House-Grete's House", "Grete's House-Farm",
"Grete's House-Shop", "Marketplace-Farm",
"Marketplace-Post Office", "Marketplace-Shop",
"Marketplace-Town Hall", "Shop-Town Hall"
];
exports.roadDesign = buildDesign(roads.map(r => r.split("-")));
